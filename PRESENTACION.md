# Prompt para generar PPT profesional

Genera una presentacion profesional en espanol para vender a un cliente una plataforma web de control laboral y asistencia.

Contexto:
La solucion es un sistema interno para una empresa que quiere centralizar el control de empleados, asistencia, permisos, vacaciones, dashboard operativo, roles de acceso, reportes y futura gestion de nominas. Ya existe una primera version funcional desplegada en Vercel, con base de datos Supabase, login privado, roles, dashboard y modulo inicial de empleados.

No presentar la solucion como terminada al 100%. Presentarla como una primera version funcional y escalable, con una base tecnica solida y un roadmap claro para completar los modulos restantes.

Objetivo de la presentacion:
Convencer al cliente de que esta plataforma reduce trabajo manual, mejora trazabilidad, ordena la informacion laboral y permite crecer hacia una gestion integral de personal.

Tono:
Profesional, corporativo, claro, sobrio y orientado a negocio. Evitar lenguaje demasiado tecnico. No usar exageraciones. Transmitir confianza, control, seguridad y escalabilidad.

Estilo visual:
Minimalista y ejecutivo. Fondo blanco o gris muy claro. Paleta sobria con azul marino, azul corporativo y grises. Usar iconos simples. Evitar colores llamativos, emojis y decoracion excesiva.

Estructura sugerida de la presentacion:

1. Portada
Titulo: Plataforma de Control Laboral y Asistencia
Subtitulo: Gestion centralizada de empleados, presencia, permisos y reportes
Incluir una frase corta: "Una base digital para ordenar la gestion diaria del personal."

2. Problema actual
Explicar que muchas empresas gestionan asistencia y permisos con Excel, WhatsApp, correos, hojas firmadas o sistemas aislados.
Mostrar consecuencias:
- Errores manuales.
- Falta de trazabilidad.
- Dificultad para consultar historicos.
- Tiempo perdido por RRHH.
- Informacion dispersa.
- Poca visibilidad para direccion.

3. Propuesta de solucion
Presentar la plataforma como un sistema web privado que centraliza:
- Empleados.
- Asistencia.
- Permisos.
- Vacaciones.
- Dashboard.
- Reportes.
- Futuras nominas visuales.

Mensaje clave:
"La plataforma convierte la gestion laboral diaria en un flujo ordenado, consultable y seguro."

4. Beneficios para direccion
Incluir:
- Vision rapida del estado de la plantilla.
- Menos dependencia de Excel.
- Datos centralizados.
- Mejor control operativo.
- Informacion preparada para reportes y decisiones.
- Plataforma escalable.

5. Beneficios para RRHH
Incluir:
- Alta y gestion de empleados.
- Control de empleados activos e inactivos.
- Seguimiento de asistencia.
- Gestion de permisos y vacaciones.
- Reduccion de tareas repetitivas.
- Historico consultable.

6. Beneficios para empleados
Incluir:
- Acceso privado con usuario y contrasena.
- Interfaz simple.
- Marcacion de asistencia.
- Consulta de informacion propia.
- Solicitud de permisos y vacaciones.
- Menos dependencia de mensajes informales.

7. Estado actual de la version funcional
Explicar con honestidad:
"Actualmente existe una primera version funcional en produccion con la base principal del sistema."

Modulos ya disponibles:
- Aplicacion web desplegada.
- Login privado.
- Roles de usuario: administrador, manager y empleado.
- Dashboard con datos reales.
- Gestion inicial de empleados.
- Base de datos en la nube.
- Repositorio y despliegue configurados.

8. Demo sugerida
Crear una diapositiva con el flujo de demostracion:
1. Iniciar sesion como administrador.
2. Mostrar dashboard.
3. Entrar al modulo de empleados.
4. Mostrar busqueda, filtros, detalle y edicion.
5. Explicar roles de acceso.
6. Iniciar sesion como empleado.
7. Mostrar que el empleado tiene acceso limitado.

9. Roadmap de crecimiento
Mostrar los siguientes modulos como fases:
- Marcacion real de entrada y salida.
- Asistencia con calculo de horas y tardanzas.
- Permisos y vacaciones con aprobacion.
- Horarios y festivos.
- Reportes exportables.
- Nominas visuales y PDF.
- Pulido final, seguridad y validaciones completas.

10. Seguridad y control
Explicar:
- No hay registro publico.
- Los usuarios son creados por administradores.
- Acceso por roles.
- Cada empleado ve solo su informacion.
- Base de datos centralizada.
- Preparado para trazabilidad y auditoria.

11. Impacto esperado
Incluir mensajes de impacto:
- Menos errores administrativos.
- Menos tiempo en tareas manuales.
- Mejor visibilidad del estado laboral.
- Informacion ordenada para decisiones.
- Mayor profesionalizacion de procesos internos.

12. Cierre
Mensaje final:
"Esta plataforma no es solo una herramienta para fichar. Es una base digital para profesionalizar la gestion del personal, reducir trabajo manual y construir informacion confiable para la empresa."

Incluir llamada a la accion:
"Siguiente paso: validar el flujo operativo con usuarios reales, priorizar los modulos restantes y avanzar hacia una version completa de produccion."

Notas importantes para la IA que genera el PPT:
- No decir que todos los modulos estan terminados.
- Diferenciar claramente "version funcional actual" y "roadmap".
- Evitar terminos demasiado tecnicos como Prisma, NextAuth, Supabase o Vercel salvo en una diapositiva opcional de arquitectura.
- Si se incluye arquitectura, mantenerla simple: Aplicacion Web + Base de Datos en la nube + Roles de acceso + Despliegue cloud.
- Mantener el texto breve por diapositiva.
- Usar graficos simples: flujo, checklist, comparativa antes/despues, roadmap por fases.
- Generar entre 10 y 12 diapositivas.
